# カスタマバーコード印字機能 設計書

**作成日:** 2026-04-27  
**対象バージョン:** v1.3.0（予定）

---

## 1. 概要

日本郵便のカスタマバーコード（4ステイト3バー）をラベルPDFに印字する機能を追加する。
バーコード割引を受けるためにはフル構成（郵便番号7桁＋住所表示番号）が必要。
印字はバッチ（データセット）単位でON/OFFを切り替えられるオプションとする。

**対象モード:** normal（氏名あり）・no_person（氏名なし）のみ  
**対象外モード:** simple・nametag・split4（住所を印字しないため意味がない）

## 4. customer_barcode.py モジュール

### 4.1 住所表示番号の抽出

```python
def extract_address_code(address: str) -> tuple[str, bool]:
    """
    住所テキストから住所表示番号を抽出する。
    Returns: (address_code, is_confident)
      address_code: "1-3-2" 形式の文字列（抽出失敗時は ""）
      is_confident: 抽出に自信があればTrue
    """
```

**対応パターン:**
- `1丁目3番2号` → `1-3-2`
- `1-3-2` / `１－３－２`（全角ハイフン） → `1-3-2`
- `1番地3号` → `1-3`
- `1番地` → `1`
- 番地が見つからない場合 → `""`, `is_confident=False`

**信頼度が低い（`is_confident=False`）条件:**
- 数字が全く見つからない
- 抽出結果が空文字
- 住所テキスト自体が空

### 4.2 バーコード構成

```
[S] + [郵便番号7桁] + [住所表示番号 最大13文字] + [チェックデジット1桁] + [T]
合計: 23文字（スタート・ストップ含む）
```

- 住所表示番号が13文字未満の場合は制御コード CC4 でパディング
- 使用文字: 数字 0-9、ハイフン(-)、制御コード CC1-CC8

### 4.3 チェックデジット計算

```python
def calc_check_digit(chars: list[str]) -> str:
    """
    全文字を数値に変換（0-9→0-9, ハイフン→10, CC1-CC8→11-18）し、
    合計が19の倍数になるよう1文字を決定する。
    """
```

計算式: `check = (ceil(total / 19) * 19) - total`  
結果が10の場合はハイフン、11-18はCC1-CC8に変換。

### 4.4 描画

```python
def draw_barcode(
    canvas,
    x0: float,    # 左端座標 (pt)
    y0: float,    # 下端座標 (pt)
    postal: str,  # 郵便番号（ハイフンなし7桁）
    addr_code: str,  # 住所表示番号
) -> None:
```

**バー寸法（a=10ポイント基準）:**
| 項目 | 値 |
|------|----|
| ロングバー高さ | 3.6mm |
| タイミングバー高さ | 1.2mm |
| バーピッチ（中心間） | 1.2mm |
| バー幅 | 0.6mm |
| バースペース | 0.6mm |

各文字を3本のバー（F/A/D/T）でレンダリング。  
F = 上下両方、A = 上のみ、D = 下のみ、T = 短いバー（タイミング）。

## 5. UI変更（DirectLabelDialog）

### 5.1 バーコードオプションチェックボックス

フッター行（件数ラベルの右）に配置:
```
0 件     [☑ カスタマバーコードを印字する]  用紙: [...] フォント: [...]  [閉じる] [PDF出力]
```

チェックON/OFFで「住所表示番号」列を即時表示/非表示。

### 5.2 住所表示番号列

| 状態 | 表示 |
|------|------|
| 自動抽出・高信頼 | 通常表示（例: `1-3-2`） |
| 自動抽出・低信頼 | 黄色背景 ＋ ツールチップで警告 |
| 手動編集済み | 通常表示（警告解除）※ユーザーがセルを編集した時点で警告フラグをクリア |
| 住所が空の行 | グレー背景・編集不可 |

- 列位置: テーブル最右列（COL_BARCODE_ADDR）
- 幅: 130px（Fixed）
- チェックOFFのとき `setColumnHidden(True)`
- チェックONのとき住所列から自動抽出して列を初期化

### 5.3 バッチ読み込み時の復元

`_load_batch()` で `barcode_enabled` / `barcode_address` を読み込み、
チェック状態と住所表示番号列を復元する。

## 6. PDF描画変更（label_pdf_service.py）

### 6.1 シグネチャ変更

```python
def generate_label_pdf(
    entries:         list,
    output_path:     str,
    batch_mode:      str  = "normal",
    layout_key:      str  = DEFAULT_LAYOUT_KEY,
    font_key:        str  = DEFAULT_FONT_KEY,
    barcode_enabled: bool = False,  # ← 追加
) -> str:
```

`entries` の各オブジェクトは `barcode_address` 属性を持つ（空文字の場合はバーコード非描画）。

### 6.2 描画位置

```
┌─────────────────────────────┐
│ 〒100-0013                  │  ← テキスト領域（バーコードON時は少し上に詰める）
│ 東京都千代田区霞が関1-3-2   │
│   株式会社○○               │
│     総務部長　山田 太郎 様  │
├─────────────────────────────┤
│▐▌▐▌▐▐▌▌▐▌▐▐▌▌▐▌▐▐▌▌▐▐▌▌▐▌│  ← バーコード（高さ約5mm）
└─────────────────────────────┘
  ↑ラベル下端から1.5mm
```

- `_draw_normal` / `_draw_no_person` に `barcode_address: str = ""` 引数を追加
- `barcode_address` が非空かつ郵便番号が有効な場合のみ描画
- バーコード高さ（3.6mm）＋上下余白（各1.5mm）= 約6.6mm をテキスト領域から引く

## 7. エラーハンドリング

| ケース | 対応 |
|--------|------|
| 郵便番号が7桁でない | バーコード非描画（エラーなし） |
| 住所表示番号が空 | バーコード非描画（エラーなし） |
| 住所表示番号が13文字超 | 13文字に切り詰め |
| バーコードが有効でないモード（simple等） | `_draw_label`内で無視 |
